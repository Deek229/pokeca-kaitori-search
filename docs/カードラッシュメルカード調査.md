## カードラッシュ

### 判定: **取得可能**

従来使っていた Google スプレッドシートは壊れたままですが、**公式の新しい買取表サイト**が動いており、そこから価格を取れます。

### これまでの問題

- `page/38`（通常弾）・`page/39`（最新弾）に埋め込まれていた **Google スプレッドシート**の公開ビューが `#REF!` になり、中身が空
- CSV 出力も `#REF!` のみ（5〜15バイト）
- そのため既存モジュールでは **0/132 件**マッチ

### 新しい公式ソース（推奨）

カードラッシュ本サイトの買取案内（`page/40`）に **「新しい買取表はコチラ」** とあり、次のサイトへ誘導されています。

| 項目 | 内容 |
|------|------|
| **URL** | https://cardrush.media/pokemon/buying_prices |
| **形式** | Next.js の Web ページ（HTML 内の `__NEXT_DATA__` に JSON） |
| **全件数** | 約 **11,900 件**（100件×119ページ） |
| **更新** | ページ上に更新日時表示あり（調査時: 06/22 17:30） |

#### データの読み方（技術メモ）

1. `https://cardrush.media/pokemon/buying_prices?page=1&limit=100` を GET
2. HTML から `<script id="__NEXT_DATA__">` の JSON を抽出
3. `props.pageProps.buyingPrices` がカード配列
4. `props.pageProps.lastPage` までページを繰り返す（調査時は **119 ページ**）

#### 1件あたりの主なフィールド

| フィールド | 例 | 用途 |
|-----------|-----|------|
| `name` | リーリエのピッピex | カード名 |
| `model_number` | 126/100 | 型番（レア版は TCGdex の番号と不一致のことがある） |
| `pack_code` | SV9 / SV9a | パックコード（セット絞り込みに使える） |
| `rarity` | SAR, RR など | レアリティ |
| `amount` | 25000 | **買取価格（円）** |
| `extra_difference` | 未開封 など | 状態メモ |

#### セット別の一括取得（テスト向け）

```
https://cardrush.media/pokemon/buying_prices?pack_code=SV9
```

- SV9 だけで **94 件**（1ページで完結）
- SV9a を足すと **135 件**

### 動かない・使えないもの

| URL / 方法 | 結果 |
|------------|------|
| Google スプレッドシート（pubhtml / CSV） | `#REF!` のみ |
| gviz CSV API | HTTP 404 |
| `cardrush-pokemon.jp/product-list` | **販売価格**（買取ではない） |
| `cardrush-pokemon.jp/kaitori/search` | HTTP 404 |
| `phone/kaitori/search` | HTTP 404 |

### SV9 マッチテスト結果

`pack_code=SV9` と `SV9a` の買取表（135件）をカタログにし、DB の SV9 全132枚と照合:

| 結果 | 件数 |
|------|------|
| マッチ成功 | **53 / 132（40.2%）** |
| マッチ失敗 | 79 |

**マッチしにくい主な理由**

- 買取表の `model_number`（例: SAR の 126/100）と TCGdex の `local_id`（例: 033）が **同じカードでも番号が違う**ことがある
- コモン・ウラノワなど **買取表に載らない低額カード**がある
- パックコードが「その他」のカードはセット一致で弾かれる

それでも **フルコンプ（34.8%）より高いマッチ率**で、実用に足る水準です。

### 実装状況

`tools/shops/cardrush.py` を **cardrush.media 優先**に更新済み（プロトタイプ）。

- 全ページ取得のため初回は **約1〜2分**（119リクエスト、0.3秒間隔）
- 旧 Google シートはフォールバックとして残しているが、現状は機能しない

## メルカード

### 判定: **困難（現時点ではほぼ不可）**

### これまでの問題

- `mercardpokemon.jp/product-list` へのアクセスが **HTTP 503**
- 既存モジュールは 0/132 マッチ

### 調査で分かったこと

#### 1. 通販サイトは「一時休業中」

503 のページ本文には次の表示があります（`data/probe3_mpoke_kaitori.html` と同内容）:

> このショップは現在一時休業中です。  
> 通販サイトのみ現在、在庫整理のため休業中です。

- **ボット対策だけでなく、サイト自体が閉じている**状態
- 店舗（秋葉原1号店）は営業中だが、**通販・Web 買取表は利用不可**

#### 2. 試した URL 一覧

| URL | 結果 |
|-----|------|
| mercardpokemon.jp（トップ / product-list / 検索） | **503**（休業ページ） |
| akihabara-cardshop.com/buylist | 404 |
| akihabara-cardshop.com/pokemon | 404 |
| akihabara-cardshop.com/kaitori | 200（**店舗買取の案内のみ**。価格表なし） |
| mercard.jp | DNS 解決不可（ドメインなし） |

#### 3. akihabara-cardshop.com について

コーポレートサイトには **ワンピース・遊戯王・デュエマ等の「高価買取表」** へのリンクが多数ありますが、**ポケモンカード用の買取表ページは存在しません**。

ポケモンは `mercardpokemon.jp`（通販）へのリンクのみ。買取価格の Web 公開はポケモンだけ別扱いで、現状は通販サイト休業の影響を受けています。

#### 4. 既存モジュールの設計上の問題

`tools/shops/mercard.py` は `product-list`（**販売商品一覧**）をパースする作りでした。たとえ 200 が返っても **買取価格ではなく販売価格**になり、本来の目的と合いません。

### SV9 マッチテスト

**0 / 132（0%）** — カタログを取得できず未実施同等
